bedhed.js

Better handling of headers for particularly long tables

a jQuery plugin by J. Ky Marsh, emotional support and comedic relief by Derek Wheelden

Ever written a particularly long HTML form or survey? One with dozens of table rows? Wouldn't it be nice if you could capture the table header cells and repeat them throughout the form automatically, saving you from having to manually repeat the cells' information, and allowing the user to see these headers at frequent waypoints throughout the form, rather than having to constantly scroll to the top?

Enter bedhed.js.

The options

bedhed.js has three different modes of operation: two subsets of the "repeater" mode, and the "fixed" mode. Each of these three modes supports differing options. Please view the table below to see which options are supported in the mode you're currently using.

mode
{"repeater", "fixed"} - Chooses mode of operation for bedhed.js. In "repeater" mode, a row of header cells will be saved and then either injected into an existing table, or cloned into a new set of tables. In "fixed" mode, a fixed-position div will be appended to the screen, floating over the table as you scroll past.
terminateTable
{true, false} - This option only works in "repeater" mode. If true, the table will be cloned and appended to the page, essentially creating a new set of tables, each with its own header row.
period
{integer} - This option only works in "repeater" mode. An integer value for the period will determine the interval at which the header row is repeated.
preserveStyle
{true, false} - This option only works in "repeater" mode when terminateTable is false. If true, the font CSS style of the header cells will be preserved for all new injected header rows.
headerClass
{"string"} - If a quoted string is passed to this option, each newly-constructed header row will receieve the string as a CSS class, in order to be able to add additional styling.

How it works

bedhed.js takes a properly-formatted table with a single header row, stores this row, and then either repeats this header or provides a fixed-positioning option, based on options you pass to it.

bedhed.js has three primary modes of operation, each with its own supported options:

Option Modes of operation
Repeated header Fixed header
terminateTable: false terminateTable: true
period (integer) YES YES NO
preserveStyle (boolean) YES NO NO
headerClass (string) YES NO YES

Notes

  • The most stable, feature-filled mode is "repeater" with terminateTable set to false. Using terminateTable is not advisable if you're using bedhed.js on an HTML form especially, as the form would be cloned and appended. Use it ONLY for static tables.
  • "Fixed" mode will not adapt to the user resizing the browser window. You'll need to refresh to update the X offset of the new div.
  • I highly recommend using a CSS class or ID with bedhed.js, not just attaching bedhed.js to the "table" jQuery-wrapped DOM element. It works best on individual tables.
  • Works great in IE7/8; doesn't appear to work in IE6. Sorry, I don't really care.
